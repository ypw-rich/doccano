    FAQ

    I can't install doccano.

    Following list is ordered by from easy to hard. If you are not familiar with Python development, please consider easy setup.

    1. One click deployment to Cloud Service.
    2. Only you have to do is create an account. Especially Heroku does not require your credit card (if free plan).

    6. Notice: (1) EC2 KeyPair cannot be created automatically, so make sure you have an existing EC2 KeyPair in one region. Or create one yourself. (2) If you want to access doccano via HTTPS in AWS, here is an instruction.

    7. Use Docker
    8. Docker doesn't bother you by the OS, Python version, etc problems. Because an environment for application is packed as a container.
    9. Get doccano's image: docker pull chakkiworks/doccano
    10. Create & Run doccano container: docker run -d --name doccano -p 8000:80 chakkiworks/doccano
    11. Create a user: docker exec doccano tools/create-admin.sh "admin" "admin@example.com" "password"
    12. Stop doccano container: docker stop doccano
    13. Re-Launch doccano container: docker start doccano
    14. Install from source
    15. I want to remember you that this is the hardest setup way. You have to install Python/Node.js and type many commands.
    16. Install Python
    17. Install Node.js
    18. Get the source code of doccano: git clone https://github.com/chakki-works/doccano.git
    19. Move to doccano directory: cd doccano
    20. Create environment for doccano: virtualenv venv
    21. Activate environment: source venv/bin/activate
    22. Install required packages: pip install -r requirements.txt
    23. Move server directory: cd app/server
    24. Build frontend library: npm install
    25. Build frontend source code: npm run build
    26. Back to server directory: cd ../
    27. Initialize doccano: python manage.py migrate
    28. Create user: python manage.py createsuperuser
    29. Run doccano: python manage.py runserver
    30. Stop doccano: Ctrl+C
    31. Re-Launch doccano: python manage.py runserver (Confirm you are at app/server directory and environment is active).

    I can't upload my data.

    Please check the following list.

    • File encoding: UTF-8 is appropriate.
    • Filename: alphabetic file name is suitable.
    • File format selection: File format radio button should be selected properly.
    • When you are using JSON/JSONL: Confirm JSON data is valid.
    • You can use JSONLint or some other tool (when JSONL, pick one data and check it).
    • When you are using CSV: Confirm CSV data is valid.
    • You can use Excel or some tools that have import CSV feature.
    • Lack of line: Data file should not contain blank line.
    • Lack of field: Data file should not contain blank field.

    You don't need your real & all data to validate file format. The picked data & masked data is suitable if your data is large or secret.
